## プロジェクト概要

### 基本情報
| 項目 | 詳細 |
|---|---|
| **プラグイン名** | Quad-Morph Filter |
| **フォーマット** | VST3 / Standalone |
| **フレームワーク** | JUCE 8 (C++20) |
| **ビルドシステム** | CMake 3.24 |
| **CPU最適化** | AVX2 SIMD (xsimd 13.0.0) |
| **ソースコード** | 3,216行（DSP + UI含む） |
| **目的** | 28種類のアナログ・デジタルフィルターの等パワーモーフィング |

### 主要機能
- **XYパッド**: 4フィルター（A/B/C/D）を2D空間上で自由に配置・モーフィング
- **28フィルターモデル**: Moog、TB-303、Oberheim、Prophet、Buchla等の著名アナログシンセ、デジタル精密フィルター
- **3系統LFO**: 19種類の波形（Sine, SAW, Pulse, 各種ランダム、Recording等）
- **モーフィング選択**: EqPwr（等パワー）/ Linear（線形）/ Smoothstep（スムーズ）/ Radial（逆距離二乗）の4アルゴリズム
- **XYモード切替**: Morph（4フィルター融合）vs Cutoff（XYでカットオフ/レゾナンス直接制御）
- **オーバーサンプリング対応**: Off / Auto / 2x / 4x の選択可能
- **Dry/Wet + ゲイン + リミッター**: マスター出力制御

## DSP処理フロー

### processBlock の実行順序

```
processBlock(buffer, midiBuffer)
  │
  1. DRY バッファ保存
  │   dryBuffer ← buffer
  │
  2. テンポ取得
  │   bpm ← PlayHead (デフォルト 120 BPM)
  │
  3. LFO エンジン処理
  │   LfoEngine::process(dt, bpm, baseX, baseY, apvts, recCtx)
  │    → 3系統LFO更新 + RecordingContext同期
  │    ← positions[3], mod4[3] 計算完了
  │
  4. XY位置取得 & モード分岐
  │   xyMode = apvts["xyMode"]
  │   ├─ mode==0: Morphモード (XYで4フィルター融合制御)
  │   └─ mode==1: Cutoffモード (XYでカットオフ/レゾ直接制御)
  │
  5. XY → Cutoff/Reso 変換（Cutoffモード時のみ）
  │   cutoffAlgo = apvts["cutoffAlgo"]
  │   ├─ algo==0: Abs絶対 (従来) → 20Hz-20kHz, Res 0.1-10
  │   ├─ algo==1: Rel相対 (中央=632Hz/1.0) → ±4oct/±2oct
  │   └─ algo==2: Zone非対称 (右+5oct/左-3oct) → 上+3oct/下-2oct
  │
  6. OSモード設定
  │   osMode = apvts["osMode"] (Off / Auto / 2x / 4x)
  │   filterA.setOsMode(osMode), filterB, filterC, filterD
  │
  7. LFO2/3 モジュレーション量計算
  │   cM = MorphEngine::computeModulation(lfo2, mod4[1], isRand1)
  │   rM = MorphEngine::computeModulation(lfo3, mod4[2], isRand1)
  │
  8. フィルターパラメータ更新
  │   for each filter A/B/C/D:
  │     model ← apvts["model{A-D}"]
  │     cutoff ← apvts["cutoff{A-D}"] + LFO2変調（Cutoffモード時）
  │     res ← apvts["res{A-D}"] + LFO3変調
  │     type ← apvts["type{A-D}"] (LP/BP/HP/Notch)
  │     slope ← apvts["slope{A-D}"] (12/24/48/96dB)
  │
  9. wMix（ウェイト） 計算
  │   if xyMode==1 (Cutoffモード):
  │     等パワー正規化: w = 1/√(有効フィルター数)
  │   else (Morphモード):
  │     morphBlend = apvts["morphBlend"] (EqPwr/Linear/Smoothstep/Radial)
  │     lfo1がRandom1なら: wMix = lfo1のmod4 そのまま
  │     else: morphBlend に従いウェイト計算 → 有効フィルターで再正規化
  │
  10. フィルター処理
  │    filterBuffers[i] ← buffer を filterA/B/C/D で処理
  │
  11. ABCD ミキシング
  │    buffer.clear()
  │    for each channel: buffer += filterBuffers[0]*wMix[0] + ... + filterBuffers[3]*wMix[3]
  │
  12. Dry/Wet ミックス
  │    mixRatio = apvts["dryWet"] / 100
  │    mixed = dry*(1-ratio) + wet*ratio
  │
  13. ゲイン適用 + リミッター
  │    gainLinear = dB→線形(apvts["masterGain"])
  │    ceilingLinear = dB→線形(apvts["limiterCeiling"])
  │    gained = mixed * gainLinear
  │    limitGain に RMS追従式でリミッター適用
  │    output = gained * limitGain

  ↓ (return to DAW)
```

### 重要なサンプリング周期

| 処理 | 周期 | 目的 |
|---|---|---|
| **LFO波形計算** | 毎サンプル | リアルタイム変調 |
| **フィルター係数更新** | 16サンプルごと | ジッパーノイズ低減 |
| **AGC（自動ゲイン補正）** | 毎サンプル | レゾナンスピークの自動補正 |
| **LFO UI同期** | 60Hz (timerCallback) | XYパッド描画・トレイル更新 |
| **オーバーサンプリング** | OS倍数に依存 | 高周波エイリアス低減（Moog/TB-303推奨2x） |

## 28フィルターモデルの実装詳細

### Model 1: Moog Ladder (12/24dB切替可能)

**特徴**: 4段ZDF ラダー構成、tanh非線形、Newton-Raphson陰的ソルバー

```
入力 → ADAA(tanh)
  ↓
4段 ZDF 各段:
  v_i = (u - s_i) * G
  y_i = tanh(v_i + s_i)
  s_i += 2*v_i
  ↓ (フィードバック: k = jmap(res, 0.1, 10, 0, 4.5))
```

**実装詳細**:
- **ADAA (Antiderivative Antialiasing)**: tanh 前サンプル入力 x_prev を保持
- **AGC**: RMS追従で共振ピークを補正 (0.1～15倍, 目標 = √(rmsIn/rmsOut))
- **自己発振シード**: ladderRes > 3.5 で 1e-5 ノイズ注入（アナログ回路の熱雑音に相当）
- **Hybrid NR**: ladderRes ≤ 4.0 は Newton-Raphson 3回反復、> 4.0 は明示的ZDF（自己発振維持）

**Type対応**: LP (12dBは y2, 24dBは y4) / BP / HP / Notch

---

### Model 2: TB-303 Diode Ladder (Accent制御)

**特徴**: 4段ZDF + 8Hz HPF、Accent (Off/Low/High) で k_max制御

```
入力 → ADAA(tanh)
  ↓
4段 ZDF (同Moog構造)
  ↓ 
8Hz HPF (固定)
  ↓
出力×0.95
```

**Accent効果**（Tim Stinchcombe / Open303 解析より）:

| Accent | k_max | カットオフシフト |
|---|---|---|
| Off | 4.2 | 1.00× |
| Low | 4.6 | 1.21× (+3.2半音) |
| High | 5.0 | 1.56× (+7.7半音) |

**重要な実装ポイント**:
- **ADAA 不適用の理由**: Stateful System（再帰的帰還フィルター）への ADAA 適用は線形ブランチ全体への同期フィルター挿入が必須であり、入力段のみへの適用は音響的に正しくない (Bilbao et al., MDPI 2020)
- **Newton-Raphson方程式**: `u_pre + k·y4_nl(tanh(u_pre)) = x` を求解
  - y4_nl: 4段 ZDF を tanh 付きで通した非線形出力
  - 連鎖律: `dy4/du_pre = G⁴·∏sech²(各段)`
- **Hybrid NR**: k ≤ 4.0 は NR 3回反復、k > 4.0 は明示的ZDF（自己発振維持）

**Type対応**: LP のみ (実機の物理特性)

---

### Model 12/13/15: Ladder系 (CEM3320/SSM2040/Jupiter)

| モデル | チップ | 入力飽和 | 各段飽和 | 特徴 |
|---|---|---|---|---|
| 12 (CEM3320) | OTA型 | 対称 tanh (1.4) | なし | Prophet-5 相当, LP/BP/HP対応 |
| 13 (SSM2040) | OTA型 | 線形 | **非対称クリップ** (偶数/奇数段逆) | Oberheim相当, LP固定, 偶数次高調波 |
| 15 (Jupiter) | OTA型 | ソフトクリップ | tanh (対称) | Roland相当, LP固定 |

**レゾナンス係数の差異**:
- CEM3320 / Jupiter: `r_scale = 4.5`
- SSM2040: `r_scale = 5.0` (同じResノブでもピークが若干高い)

**EQカーブの同一性**: 3モデルとも同じ4-pole ZDF ラダートポロジーのため、線形領域（EQアナライザー）での周波数応答は同じ形。差は **非線形域** での音色（高調波内容）に現れる。

### Model 0: Clean SVF (12/24/48/96dB段階選択)

**特徴**: 2-pole SVF を 1/2/4/8 段カスケード可能（他のカテゴリから異なる）

```
1-pole SVF:
  g = tan(π*fc/fs)
  k = 1/Q
  v3 = x - v2 - k*v1
  v1_new = v1 + g*h*v3
  v2_new = v2 + g*v1_new
  
  LP = v2, BP = v1, HP = x - k*v1 - v2, Notch = x - k*v1
```

**SIMD対応**: FilterA_SVF_SIMD で AVX2 並列処理（ただし processBlock では TptFilter 経由で処理し、スロープが正しく反映されるようになっている）

**Type対応**: LP / BP / HP / Notch

### Model 3-9: SVF系フィルター

| Model # | 名称 | 特性 |
|---|---|---|
| 3 | SEM (Oberheim) | 2-pole固定 / Slope=12/24dB（ダブルSEM拡張） |
| 4 | Bitcrush / SRR | バイクラッシュ＆サンプルレート低減の融合 |
| 5 | Formant (Vowel) | 3-pole SVF ×3 並列（フォルマント） |
| 6 | Comb Filter | 遅延ライン＋フィードバック（遅延時間は smoothedDigitalCutoff で決定） |
| 7 | MS-20 (Screaming) | 4-pole SVF ＋ DC ブロッカー（フィードバック経路） |
| 8 | Phaser | All-Pass 16段 ＋ フィードバック |
| 9 | Wavefolder | sin(x) エクスペンション ＋ 1次 ADAA |

**Model 4 (Bitcrush/SRR)の実装最適化**:
- 従来: `std::pow(2, bits)` を `processSample()` 内で毎サンプル計算 → **音声スレッド違反**
- 修正: `updateCoefficients()` で `srrBitSteps = 2^bits` を事前計算してキャッシュ → processSample では乗算のみ

**Model 6 (Comb Filter)の改善**:
- 従来: 遅延時間を `currentCutoffVal`（スムージングなし）で計算 → ジッパーノイズ
- 修正: `smoothedDigitalCutoff` を使用 + ループ外で `delaySamples, eta, fb` を事前計算

**Model 7 (MS-20)のDCブロッカー**:
- フィードバック経路にキャパシタを追加（実機 MS-20 相当）
- DC成分の蓄積を防止（非対称 tanh が生むDCドリフト対策）

**Model 9 (Wavefolder)のADAA**:
- 波形: `f(x) = sin(x)`、不定積分: `F(x) = -cos(x)`
- **double 精度必須**: float では特異点付近で量子化ノイズが可聴域に現れる

### Model 10: FDN Reverb (Room/Hall/Cave/Plate)

**特徴**: 4-delay Feedback Delay Network（全面再設計 v1.0.0）

```
FDN構成:
  4つの遅延ライン（Schroeder型相互配置）
  ↓
  Per-delay LP フィルター（ダンピング補正）
  ↓
  オートコリレーション混合出力
```

**Slope別キャラクター**: (型は無視, Slope で制御)
- Slope 0 = Room（小さい，反射速い）
- Slope 1 = Hall（中程度）
- Slope 2 = Cave（大きい，尾長い）
- Slope 3 = Plate（スプリング状，独特の金属質）

**実装詳細**:
- `fdnLpState[4][2]`: Per-delay LP 状態変数（高周波吸収, 実空間の素材特性を模倣）
- `fdnBuffer[4][2][16384]`: 最大16384サンプルの遅延ライン
- Loop型で無音状態から自動発振しない（明示的 ZDF 近似）

### Model 11: Kilo All-Pass

**特徴**: 16段 All-Pass フィルター（高精度位相処理）

```
各段の All-Pass:
  y = -x + 2*(x*g + s1) / (1 + g*(g + 2))
  s1_new = s1 + g*(x*g + s1)
```

**用途**: Ultra-高周波フェージング、ウィデオ分散、デジタル音色加工

### Model 17-20: Digital Precision

| Model # | 名称 | 応答特性 |
|---|---|---|
| 17 | Butterworth | 最大平坦（リップルなし） |
| 18 | Chebyshev | パスバンドリップル有（急峻カットオフ） |
| 19 | Bessel | 線形位相保証（パルス歪小） |

## LfoEngine（19波形）

### 構造

```cpp
struct LfoState {
  float phase;
  Point currentRandom, nextRandom;     // Random 2
  array<float, 4> currentRand1, nextRand1;  // Random 1
  float bilX, bilY, bilVx, bilVy;      // Billiard
  float smoothNx, smoothNy, tNextNx, tNextNy;  // Smooth Noise
};
```

### 19波形一覧

| # | 波形 | 特徴 | 出力形式 |
|---|---|---|---|
| 0 | Sine | 標準正弦波 | (x,y) = (sin(p), sin(p+π/2)) |
| 1 | SAW | ノコギリ波 | (x,y) = 位相シフト版 |
| 2 | Pulse | パルス波（±1） | (x,y) = 位相シフト版 |
| 3 | Random 1 | Per-filter独立ランダム | mod4[i] 直接返却 |
| 4 | Random 2 | 線形補間ランダム | (x,y) = (currentRnd + (nextRnd-currentRnd)*t)*2-1 |
| 5 | Noise | 毎サンプルランダム | (x,y) = (rng*2-1, rng*2-1) |
| 6 | Recording | XY軌跡記録再生 | (x,y) = recBuffer[len-1] または 補間 |
| 7 | Smooth Noise | Hermite補間ノイズ | smoothstep カーブで平滑化 |
| 8 | Spirograph | 回転放物線 | x=0.5cos(p)+0.5cos(5.2p), y=0.5sin(p)-0.5sin(5.2p) |
| 9 | Harmonic Swarm | 3周波倍音クラスタ | (x,y) = (sin群/3, cos群/3) |
| 10 | 3D Torus Knot | トーラス結び目 | r=cos(2p)+2, x=r*cos(3p)/3, y=r*sin(3p)/3 |
| 11 | Lissajous | リサージュ曲線 | (x,y) = (sin(3p), sin(4p)) |
| 12 | Spiral | 回転スパイラル | (x,y) = (t*cos(5p), t*sin(5p)) |
| 13 | Star | 星形パターン | (x,y) = (cos(5p)*sin(t), sin(5p)*sin(t)) |

## モーフィングエンジン（4アルゴリズム）

### computeEqualPowerWMix (EqPwr)

```cpp
angleX = x * π/2
angleY = y * π/2
wA = cos(angleX) * cos(angleY)
wB = sin(angleX) * cos(angleY)
wC = cos(angleX) * sin(angleY)
wD = sin(angleX) * sin(angleY)
```

**特性**: 常に `wA² + wB² + wC² + wD² = 1` → 中央でも音量一定（-3dB）

**用途**: 最もスタンダード、滑らかな遷移

---

### computeLinearWMix (Linear)

```cpp
wA = (1-x) * (1-y)
wB = x * (1-y)
wC = (1-x) * y
wD = x * y
```

**特性**: バイリニア補間、中央で wA=wB=wC=wD=0.25 (-12dB)

**用途**: シンプルで倍音変化がダイレクト

---

### computeSmoothstepWMix (Smoothstep)

```cpp
sx = smoothstep(x) = x²(3 - 2x)  // S字カーブ
sy = smoothstep(y) = y²(3 - 2y)
// 後に EqPwr 計算
```

**特性**: コーナーへの「引き寄せ感」が強い、意図したフィルターに素早く収束

**用途**: 境界での動作をシャープにしたい場合

---

### computeRadialWMix (Radial / 逆距離二乗)

```cpp
wA = 1 / (x² + y² + ε)
wB = 1 / ((1-x)² + y² + ε)
wC = 1 / (x² + (1-y)² + ε)
wD = 1 / ((1-x)² + (1-y)² + ε)
// sum で正規化
```

**特性**: 最も近いコーナーが支配的、境界で急峻な切替

**用途**: 明確な4分割感、対角線上でも2つのフィルターのみ存在感

## UI/XYパッド座標系統一

### 過去の課題と修正

**根本原因**: paint() でのドット表示とラベル位置、updatePosition() の座標解釈が食い違っていた。

**具体例**:
- ラベル "C" は `(10, height-30)` に描画 → 視覚的中心は `(20, height-20)`
- 旧 updatePosition: `x_norm = e.x / getWidth()` → x=0 は端ピクセル
- 結果: ラベル "C" でも `wC ≈ 0.988`（完全な100%にならない）

### 修正後の座標系設計

```cpp
// コーナー 100% 点 = ラベル中心
A: (20, 20)         B: (width-20, 20)
C: (20, height-20)  D: (width-20, height-20)

// 正規化 ↔ ピクセル 変換
toNorm(px, py):
  x_norm = (px - 20) / (width - 40)
  y_norm = (py - 20) / (height - 40)

toPix(nx, ny):
  px = 20 + nx * (width - 40)
  py = 20 + ny * (height - 40)
```

### 修正成果

✅ ラベルの場所 = そのフィルターが 100%  
✅ ドット表示位置とマウス操作が完全に一致  
✅ Recording モードのパス記録も同じ座標系を使用 → LFO も正しく動作  
✅ jlimit により枠外ドラッグは枠端に吸着  

### UI同期処理 (XYPadComponent::timerCallback)

毎60Hz (timerCallback):
1. LFO位置を trails[] に記録 → トレイル描画
2. Cutoffモード時: XY位置をカットオフ/レゾスライダーに書き戻し

## リアルタイム安全性への対応

### CLAUDE.md の絶対制約遵守状況

#### 1. 動的メモリ確保の禁止 ✅

**processBlock 内でのメモリ割り当てなし**:
- `prepareToPlay` で全状態変数を事前確保
- SVF状態: `s1[8][2]`, `s2[8][2]` (スタック)
- Ladder状態: `zdfState[8][2][4]` (スタック)
- Comb遅延: `combBuffer[8][2][16384]` (スタック)
- FDN遅延: `fdnBuffer[4][2][16384]` (スタック)

**JUCE SmoothedValue の使用**:
- `juce::SmoothedValue<float>` で カットオフ/レゾ スムージング
- `reset()` で初期化、processBlock では `getNextValue()` のみ呼び出し

#### 2. ブロッキング・ロックの禁止 ✅

**ロックフリー・スレッド通信**:
- `std::atomic<int>` で recLength[3], isWaitingForRecord[3] 等
- `std::atomic<float>` で currentRecX[3], currentRecY[3]
- `memory_order_acquire/release` で明示的メモリ順序指定
- UI（メインスレッド）→ Audio（リアルタイムスレッド）: 読み取り専用

#### 3. システムコール禁止 ✅

**音声処理スレッド内の禁止事項を遵守**:
- ❌ `std::cout`, `printf` ログ出力（デバッグ用も禁止）
- ❌ `fopen`, ファイルI/O
- ❌ `juce::MessageManager::isThisTheMessageThread()` の blocking 呼び出し
- ✅ パラメータ読み込み: `apvts.getRawParameterValue()->load()` (atomic, 安全)

#### 4. UIコンポーネント操作禁止 ✅

**音声スレッドから UI 描画指示なし**:
- UI更新は `XYPadComponent::timerCallback()` (メインスレッド) で実施
- Audio → UI データフロー: `public std::atomic<>` メンバ経由のみ

### デノーマル値対策 ✅

```cpp
// processBlock 冒頭で宣言
juce::ScopedNoDenormals noDenormals;
```

## 既知の課題と改善方針

### 課題1: Model 12, 13, 15 (Ladder系) の EQカーブ同一性

**現象**: EQ アナライザーで 3モデルが全く同じカーブを示す

**原因**: 4-pole ZDF ラダートポロジーが共通 → 線形領域（周波数応答）は同じ

**解決方法**: 音色の差は **非線形域**（レゾナンス駆動時）で現れることを確認済み
- Model 12: 奇数次高調波（Prophet-5 的）
- Model 13: 偶数次高調波（Oberheim 的ウォーム）
- Model 15: 柔らかい tanh 飽和（Jupiter 的）

→ **設計上正常**。DAW内では周波数特性グラフには映らず、耳で聴くと異なる。

---

### 課題2: TB-303 Slope コンボの表示名変更

**現況**: UI では "12dB/oct" 等が表示されているが、Model 2 では Accent (Off/Low/High) として機能

**実装状況**: ModelCapabilities.h で制御
```cpp
case 2: return { 2, true, false, false, false };  // maxSlopeIdx=2
```

**改善案**: UI レベルで Model 2 選択時に Slope ラベルを "Accent" に変更（実装待ち）

---

### 課題3: Moog/TB-303 オーバーサンプリング推奨

**推奨**: OS倍数 = 2x （より高い精度が必要な場合は 4x）

**理由**: Model 1/2 の高周波エイリアス低減（tanh 非線形）

**現況**: osMode パラメータで Auto/2x/4x 選択可能。Moog/TB-303 で Auto を選ぶと 2x に自動スケーリング。

---

### 課題4: FilterA_SVF_SIMD の未完全化

**現況**: Step 1 (AoS基盤) で実装済み。processBlock で無効化（全モデルを TptFilter 経由で処理）

**理由**: CleanSVF も Slope（ステージ数）が正しく反映される必要があり、SIMD版では対応不可

**改善案**: Step 2 (SoA + AVX2化) でコンパイル時選択できるようにする
